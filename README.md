# 🛫 webwing · 타임인아웃 근태 자동 조종석

API도 MCP도 없는 **타임인아웃(근태 SaaS)** 을 브라우저 자동화로 감싸, 로그인부터 데이터 수집·분석·시각화까지 대신 해주는 도구.

> "API를 안 주는(못 주는) 서비스는, 자동화가 곧 그 서비스의 API가 된다."

본인 계정으로 로그인하면 → 가상 브라우저가 대신 타임인아웃에 접속해 **내 근태를 긁어와 → 초과근무를 계산해 → 리포트로** 보여준다. 관리자 권한 불필요.

## 핵심 기능

- **직원(본인) 모드** — `user.timeinout.kr` 로그인만으로 내 근태 조회 (관리자 권한 X)
- **찐 퇴근 기준 초과근무** — 회사의 "정량 12시간 인정 상한"을 무시하고 **실제 출퇴근 펀치**로 계산
- **월 52시간 체크** + KPI: 총 근로 / 초과근무(평일 8h 초과) / 휴일근무(예외 별도 집계)
- **출퇴근 시각 분포 그래프** — 하루를 `근무 4h → 점심 1h → 근무 4h → 저녁 30m → 야근`으로 분할, 자정 넘는 퇴근은 24h 위로
  - 색: 근무(파랑) · 야근(빨강) · 휴일근무(보라) · 휴가(초록) · 정정(청록) · 기록누락(회색)
- **자동 분류** — 주말/공휴일, 연차·반차, 정정(수동수정) 내역, 미체크아웃/한쪽만 찍힘/결근 = 기록누락
- **가상 사파리 스냅샷 플레이어** — 로그인 → 수집 → 리포트를 **실시간 스트리밍**으로 재생 (‹ › 앞뒤 탐색, ↻ 다시하기)

## 어떻게 동작하나

```
브라우저(Playwright) ──▶ user.timeinout.kr 로그인
                     ──▶ 나의 근태(일별 카드) 스크래핑  ┐
                     ──▶ 나의 휴가 내역 스크래핑        ├─▶ 분석(초과근무·분류) ─▶ 웹 대시보드
   각 단계 스냅샷 캡처 ──────────────────────────────┘         (Chart.js)
        └▶ NDJSON 스트림으로 프론트에 실시간 전송
```

- **엔진** `src/lib/timeinout.mjs` — 로그인/스크래핑/분석 (관리자·직원 모드 공용 계산부 `buildDays`)
- **서버** `server.mjs` — Express, `/api/overtime/stream` (스냅샷 실시간 스트리밍)
- **UI** `public/index.html` — 가상 브라우저 + 스냅샷 플레이어 + 대시보드 (의존성 없이 단일 파일 + Chart.js CDN)

## 설치 & 실행

```bash
npm install
npx playwright install chromium      # 헤드리스 크롬 (~150MB)

# 자격증명 (선택 — UI에서 직접 입력해도 됨)
cp .env.example .env && vi .env       # TIMEINOUT_ID / TIMEINOUT_PW

npm run dev                           # → http://localhost:8181
```

브라우저에서 열고 아이디·비번·월 넣고 **`로그인하고 자동 분석 ▶`**. `.env`에 값을 넣어두면 자동 프리필된다.

## 참고 / 한계

- **크롬(Playwright)이 필수** — 진짜 브라우저를 띄워 조작하므로 실행 환경이 묵직하다(요청당 헤드리스 크롬 1개, ~20–30초). 개인·소규모용. 크롬 인스턴스는 재사용한다.
- 자격증명은 로컬 `.env` 에만 저장되며 **git에 올라가지 않는다**(`.gitignore`). 화면 프리필도 서버가 `.env`에서 읽어오는 방식.
- 계산 전제: 휴게 90분(점심60+저녁30) 일괄 차감, 주말/휴일 근로는 전체를 초과로 집계, 미체크아웃 의심(>16h)·결근·한쪽만 찍힘은 "기록누락"으로 합계에서 제외.
- 타임인아웃 화면 구조가 바뀌면 셀렉터를 손봐야 할 수 있다.
- 자동화 사용은 대상 서비스 약관을 확인하고 본인 계정·정당한 용도로만.
